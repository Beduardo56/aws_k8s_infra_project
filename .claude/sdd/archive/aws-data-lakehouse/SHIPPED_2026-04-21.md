## Summary

A fully destroyable, serverless AWS data lakehouse — provisioned by Terraform — that ingests simulated energy-meter telemetry from a local Python producer via Kinesis Firehose, lands it in an S3 Bronze bucket, transforms it through an AWS Glue 5.0 PySpark job into an Apache Iceberg Silver table registered in the Glue Catalog, and exposes three Athena Gold views (`v_hourly_energy`, `v_daily_device_summary`, `v_fleet_daily_rollup`). Guarded by AWS Budgets + SNS email alerts, an Athena workgroup scan-cap, a Glue DPU/timeout cap, and S3 lifecycle rules — the entire 7-day demo is projected to cost ~$2.15, leaving $7.85 of headroom against the hard $10 USD ceiling. Repo is structured as a portfolio-grade showcase with six capability-oriented Terraform modules, a Mermaid architecture diagram, sample Athena queries, and a GitHub Actions workflow that runs `terraform fmt -check` and `validate` on every PR.

## Lessons Learned

### Process

- **The full 5-phase SDD flow (brainstorm → define → design → build → ship) genuinely paid off on an infra project.** The brainstorm's budget-math-during-validation caught the hourly-Glue cost overrun *before* any HCL was written, saving a rewrite of the EventBridge cadence. Cost-aware brainstorming should become a default step for any AWS project that has a hard spend ceiling.
- **Validating critical assumptions at the start of DESIGN (not the end) was well worth the ~15 min.** Three of the ten assumptions needed adjustment (Firehose event-time partitioning, Iceberg table declaration approach, destroy ordering). Discovering these at design time, not build time, prevented HCL churn.
- **Defer unit tests when E2E is the authoritative signal.** The producer has ~100 lines; unit tests for `chunked()`/`iter_records()` would have added overhead without real assurance, because all the failure modes live in `boto3` / Firehose / Glue — which unit tests can't reach. E2E manual ATs on live AWS are the right first line of defense here.
- **Single-session end-to-end is viable for a well-scoped portfolio project.** The entire brainstorm → ship loop ran in roughly two calendar days, largely one working session, *because* scope was aggressively YAGNI'd in brainstorm (16 features deferred).

### Technical

- **Iceberg table declaration via Terraform is a trap on AWS Provider 5.x** — `aws_glue_catalog_table` with `open_table_format_input.iceberg_input` is supported but limits you to identity partition transforms; `bucket(N, col)` and `days(col)` cannot be expressed there. Creating the table via `CREATE TABLE IF NOT EXISTS` from the Glue job + a destroy-time `aws glue delete-table` hook is cleaner and gives full Iceberg-spec expressiveness.
- **The "empty Bronze → views that reference non-existent Silver table" chicken-and-egg is solved by a synchronous `null_resource.initial_run`** at apply time. Cost: ~$0.07 per apply and ~5 minutes of apply duration. Upside: one-command deploy stays honest.
- **HCL `$` escape in Budgets `cost_filter` values is genuinely confusing.** `"user:Project$${var.project_name}"` looks right but produces a literal `${var.project_name}`. Use `format("user:Project$%s", var.project_name)` to make intent unambiguous.
- **Delivery-time vs event-time partitioning is a hygiene decision, not a correctness decision**, when Silver re-partitions via a derived column. Skipping Firehose Dynamic Partitioning saves cost, surface area, and Lambda plumbing — the Glue job's `measured_date` derivation is the real source of truth.
- **`force_destroy = true` on every S3 bucket is load-bearing for the "destroy on demand" requirement.** Forgetting it on even one bucket blocks the whole teardown.
- **Glue 5.0 + `--datalake-formats iceberg` is significantly simpler than Glue 4.0 + `--extra-jars`.** No JAR upload, no version-drift risk, Python 3.11 matches the producer out of the box.
- **Athena workgroup scan cutoff is a cheap, highly effective budget guardrail.** 1 GiB = $0.005/query worst case. Set it once, forget it.
- **Spark `spark.read.json(paths)` must have `ignoreMissingFiles = true` set when paths may not exist yet** — crucial for the first-run pattern. Otherwise the initial Glue run fails with `InvalidInputException: No Files Matched`.

### Communication

- **Leading with a recommendation + budget math made discovery questions fast.** Each of the 5 questions presented a ranked option table with cost/expertise signals — user answered "a" on all five without needing to research.
- **The `$10 ceiling` as a hard constraint up front was the single most useful piece of framing.** It disqualified EMR-on-EC2 instantly, which would otherwise have consumed a full brainstorm round.
- **"Portfolio-grade" as an adjective was ambiguous and needed breaking down in the last question** (minimal / portfolio-grade / overkill repo scope). Defining tiers prevented over-engineering.

### Tools & Libraries

- **AWS Glue 5.0 + Iceberg 1.6 bundled connector** — the "right default" for serverless Spark+Iceberg today; simplest possible config.
- **Terraform `null_resource` with `local-exec` + destroy-time provisioner** — unglamorous but reliable for imperative side-effects that don't have first-class TF resources (creating Athena views, dropping Iceberg tables, seeding data).
- **`random_id` for bucket-name uniqueness** — cleaner than putting account IDs in variables; keeps the repo account-agnostic.
- **`default_tags` on the AWS provider** — propagates `Project=...` tag to every resource with one line; critical for Budgets cost_filter, Cost Explorer grouping, and cleanup.
- **Firehose Direct PUT** — at this volume (~15 MB/day), it's effectively free and eliminates the Kinesis Data Streams consumer/producer complexity.
- **Existing `fake_data_generator` package** — reusing it verbatim (with `sys.path.insert`) was trivial and kept the producer under 110 lines.

## Acknowledgments

Two design moments saved hours of wasted work:

1. **Decision 3 pivot during Build** — originally planned to declare the Iceberg Silver table via `aws_glue_catalog_table` with `open_table_format_input`. Implementation revealed the `bucket(8, device_id)` transform couldn't be expressed declaratively, so the approach pivoted to "Glue job creates the table + destroy-time hook drops it" — cleaner, less brittle, and fully captured in the BUILD_REPORT's Deviations section.

2. **Brainstorm budget math caught hourly Glue cadence before any HCL existed** — the original Q4 answer ("EventBridge hourly") would have produced ~$12 Glue spend on a $10 budget. The Validation Checkpoint 1 cost-breakdown table flagged it, the user approved the every-3-hour pivot, and the final projection came in at $2.15 for 7 days. Cost math belongs *in* the brainstorm, not after DESIGN.
